# Workflow: Architectural Development (Qwen Architect)

Principle: **Never plan complex features alone. Always use a trusted Architect.**

1. **Gather Context:**
  - Use `qwen_list_files` and `qwen_read_file` to gather context from the repository if you don't have full visibility of the structure.
2. **Precise Goal:**
  - Define the `goal` field as clearly as possible, with maximum business intent based on ROI, as requested by the user.
  - Optionally collect and include `context` (e.g., legacy system code, integration outlines).
3. **Tool Invocation:**
  - Run `qwen_architect`.
4. **Blueprint Implementation:**
  - Analyze the proposed schematic (Blueprint) generated by Qwen with the user.
  - After approval - break the project down into smaller execution iterations and pass them to Generation tools (`qwen_coder`).
  - Update `.md` files (e.g., `LOGBOOK.md` and `TODO.md`), documenting the architectural outline to maintain long-term continuity.
